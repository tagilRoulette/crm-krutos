## 1. Модели данных (сущности)

В проекте используются следующие кастомные модели данных (сущности) СУБД PostgreSQL, связанные каскадными ограничениями на уровне репозиториев:

### ProjectEntity (Проект CRM)
Хранит общую конфигурацию и метаданные создаваемых пользователями CRM-систем. Является родительской сущностью верхнего уровня.
**Основные поля:**
- `Id` — уникальный идентификатор проекта (Тип: `Guid`, Primary Key).
- `Name` — пользовательское наименование конфигурации CRM (Тип: `string`, лимит 200 символов).
- `NavigationType` — тип глобального меню готовой системы (Тип: `int` / Enum: `0` — `Top`, `1` — `Side`).
- `CreatedAt` — дата и точное время инициализации проекта в формате UTC (Тип: `DateTime`).

---

### PageEntity (Страница проекта)
Описывает изолированные разделы или вкладки внутри конкретной CRM-системы. Связана с моделью проекта отношением «один-ко-многим» (1:N).
**Основные поля:**
- `Id` — уникальный идентификатор страницы (Тип: `Guid`, Primary Key).
- `Name` — заголовок страницы, отображаемый в меню (Тип: `string`).
- `CreatedAt` — временная метка создания страницы на сервере (Тип: `DateTime`).
- `ProjectId` — внешний ключ, ссылающийся на родительский проект (Тип: `Guid`, ForeignKey с `OnDelete: Cascade`).

---

### ElementEntity (Компонент/Виджет холста)
Представляет собой интерактивный элемент интерфейса, размещаемый на страницах конструктора. Связан со страницей отношением «один-ко-многим» (1:N).
**Основные поля:**
- `Id` — уникальный идентификатор конкретного виджета (Тип: `Guid`, Primary Key).
- `Json` — сериализованный слепок динамических параметров (координаты X/Y, размеры, цвета, типы компонентов: кнопки, таблицы, формы) (Тип: `string` / `jsonb`).
- `LastModified` — дата последнего изменения геометрии или свойств виджета на холсте (Тип: `DateTime`).
- `PageId` — внешний ключ, привязывающий элемент к конкретной вкладке (Тип: `Guid`, ForeignKey с `OnDelete: Cascade`).

## 2. Ключевой функционал и Use Case сценарии

### Пользовательские сценарии (User Stories / Use Cases)

#### Сценарий 1: Управление проектами CRM и развертывание шаблонов
1. **Пользователь** открывает главную страницу конструктора (MainPage).
2. **Система** через REST API контроллер `ProjectsController` делает выборку всех существующих проектов из базы данных.
3. **Пользователь** нажимает кнопку «Создать проект из шаблона».
4. **Сервер** вызывает метод `CreateTemplateProjectAsync`, считывает зашитые JSON-файлы заготовок, создает сущность проекта и наполняет её дефолтными страницами и элементами.
5. Интерфейс динамически обновляется, отображая плитку новой CRM.

#### Сценарий 2: Многопользовательское проектирование на интерактивном холсте
1. **Пользователь №1** и **Пользователь №2** одновременно открывают страницу `ConstructorPage` одного и того же проекта.
2. Клиентское React-приложение инициализирует полнодуплексный WebSocket-канал связи с `CrmConstructorHub` на бэкенде.
3. **Пользователь №1** перетаскивает виджет таблицы (Drag-and-Drop) на новые координаты.
4. В момент отпускания курсора фронтенд отправляет новое состояние по сокетам в метод `SaveElementPositionAsync`.
5. **Система** осуществляет широковещательную рассылку (Universal Broadcast) через `Clients.Others`. На экране **Пользователя №2** виджет мгновенно и без перезагрузки страницы перемещается в новую позицию.

#### Сценарий 3: Кастомизация свойств компонентов и фоновое сохранение
1. **Пользователь** кликает на виджет кнопки на холсте и открывает панель свойств (Properties Panel).
2. Изменяет цвет кнопки и внутренний текст.
3. Состояние UI-компонентов мутирует в стейт-менеджере на клиенте.
4. Измененный JSON-пакет асинхронно летит на бэкенд, где `ElementsService` выполняет сквозную валидацию ключа страницы `PageId` и обновляет поле `Json` в таблице `ElementEntity` СУБД PostgreSQL.
